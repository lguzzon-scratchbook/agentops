# Finding: Duplicated contract scanners drift when a new accepted syntax is added to one validator without paired fixture coverage in the mirror validator.

## Summary
Duplicated contract scanners drift when a new accepted syntax is added to one validator without paired fixture coverage in the mirror validator.

## Pattern
Duplicated contract scanners drift when a new accepted syntax is added to one validator without paired fixture coverage in the mirror validator.

## Detection Question
When a CI gate and smoke test both enforce the same contract, did this change update every recognizer and add a fixture for the newly accepted syntax form?

## Checklist Item
For mirrored validators, add or update paired fixtures for each accepted syntax representation and cite both validation commands in the PR proof.
