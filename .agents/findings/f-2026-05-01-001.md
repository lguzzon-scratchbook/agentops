# Finding: Default-flip atoms gated on a forward-looking soak window must also document the rollback procedure for the post-gate window — the gate is forward-looking, not backward-looking, so a regression discovered after the soak closes leaves no rescue path.

## Summary
Default-flip atoms gated on a forward-looking soak window must also document the rollback procedure for the post-gate window — the gate is forward-looking, not backward-looking, so a regression discovered after the soak closes leaves no rescue path.

## Pattern
Default-flip atoms gated on a forward-looking soak window must also document the rollback procedure for the post-gate window — the gate is forward-looking, not backward-looking, so a regression discovered after the soak closes leaves no rescue path.

## Detection Question
Does the plan describe what happens if a regression is found AFTER the soak window closes and the legacy code path has been deleted?

## Checklist Item
Verify atom-5-style default-flip atoms have a 3-step rollback procedure (rapid response, code rollback, spec rollback) in their issue body and reference foundation §9 escalation contract.
