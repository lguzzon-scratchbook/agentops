# Finding: When a heavy fitness goal depends on multi-session corpus state, repeated failures across nightlies are the gate working correctly while the upstream input is empty. Quarantine via weight reduction is partial; a corpus-active precondition (skip when total citations in window == 0 and total_artifacts > 0) is durable.

## Summary
When a heavy fitness goal depends on multi-session corpus state, repeated failures across nightlies are the gate working correctly while the upstream input is empty. Quarantine via weight reduction is partial; a corpus-active precondition (skip when total citations in window == 0 and total_artifacts > 0) is durable.

## Pattern
When a heavy fitness goal depends on multi-session corpus state, repeated failures across nightlies are the gate working correctly while the upstream input is empty. Quarantine via weight reduction is partial; a corpus-active precondition (skip when total citations in window == 0 and total_artifacts > 0) is durable.

## Detection Question
When a goal has failed N consecutive nightlies with the same root cause (corpus dormancy, σ=0 ρ=0), did the goal definition or measurement script include a precondition that distinguishes 'corpus dormant' (skip with reason) from 'corpus active but underweight' (real fail)?

## Checklist Item
For corpus-state goals, add a precondition: if upstream corpus has zero citation activity in the measurement window AND total_artifacts > 0, return SKIP with reason 'corpus-dormant; gate cannot be moved by single-session work' rather than FAIL.
