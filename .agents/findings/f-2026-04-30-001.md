# Finding: Dream curator emits packets the inline-probe rejects as stale, with the same packet recurring across nightlies.

## Summary

Dream's morning-packet curator selected 3 packets this run (`nightly/2026-04-30`); 2 (66%) failed the nightly's 5-second inline tractability probe — well above the 30% MUST-spend threshold and the 50% finding threshold. Per-packet `dream-corpus-stale` notes are no longer informative; the same packet (`dream-df960e173c38bb56` "Decompose skills/crank/SKILL.md to under 248-line limit") was emitted on two consecutive nightlies and probed stale both times. Escalating to curator-level.

## Pattern

The curator's `probeDreamPacketStaleness` (`cli/cmd/ao/overnight_packets.go:195`) only checks two narrow surfaces:

1. Literal `TargetFiles` entries that exist on disk → treated as already done.
2. `scripts/<x>.sh` tokens extracted from `MorningCommand` or `Title` via `extractScriptsRef`.

This misses two large classes of stale packets:

- **`skills/<name>/SKILL.md` tokens.** Packet `dream-df960e173c38bb56` cites `skills/crank/SKILL.md` (which exists) but the title claims a fictional 248-line limit. `scripts/check-skill-size.sh` actually warns at 500 and fails at 800; `skills/crank/SKILL.md` is 660 (WARN, not FAIL). The probe doesn't extract the skill ref so doesn't run any check.
- **Already-shipped fixes by name.** Packet `dream-476e66319e8d18d3` (eval-advisory) was fixed by merged PR #172 (commit `67a783ac fix(ci): install eval advisory dependencies`) and marked consumed in still-open triage PR #176. The packet ID matches a `next-work.jsonl` item that lives on `triage/2026-04-29` (unmerged) as consumed; today's main has it as `available`. The probe doesn't cross-check item IDs against open triage branches.

## Detection Question

Before emitting a morning packet, did the curator probe (a) every cited file/symbol/script — including `skills/<x>/SKILL.md` and `schemas/*.json` — and (b) cross-check the source item against any open triage PR's consumed marks for the same item ID?

## Checklist Item

Extend `probeDreamPacketStaleness` to:

1. Detect `skills/<name>/SKILL.md` tokens in title/command and treat as stale when the file is below the actual size-check thresholds (warn=500, fail=800) — or otherwise extant for an "add"-style packet.
2. Detect `schemas/<x>.json` tokens analogously to `scripts/<x>.sh`.
3. Cross-check `item.ID` (or normalized `(source_epic, title)` tuple) against open triage PRs' `next-work.jsonl` consumed marks before emit.

Two consecutive nightlies emitting the same conclusively-stale packet is the trigger.
