# Planning Rule: Task-only PR queue closures can have strong merge-commit evidence but still fail child-oriented closure-integrity collection because there are no child beads to inspect.

Prevent this known failure mode:

- Pattern: Task-only PR queue closures can have strong merge-commit evidence but still fail child-oriented closure-integrity collection because there are no child beads to inspect.
- Ask: When post-morteming a queue-drain task rather than an epic, did the closure proof cite merge commits or an evidence-only packet before invoking child-oriented closure checks?
- Do: For task-only queue drains, record merge commit evidence or generate a durable evidence packet, and treat missing child beads as WARN rather than FAIL when task proof is present.
