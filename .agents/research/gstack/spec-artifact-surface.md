## Templated Artifacts (One-Source-Many-Hosts)

Every skill ships a `.tmpl` source. `bun run gen:skill-docs` materializes one `SKILL.md` per host into the skill dir; per-host overrides land in `<skill>/<host>/SKILL.md` (e.g., `codex/SKILL.md` is the codex-specific render of the root template).

| Source | Generator | Output |
|---|---|---|
| `<skill>/SKILL.md.tmpl` | `scripts/gen-skill-docs.ts` (`bun run gen:skill-docs`) | `<skill>/SKILL.md` per host |
| `SKILL.md.tmpl` (root) | same | `SKILL.md` (root umbrella skill) |
| `claude/SKILL.md.tmpl` | same with `--host claude` | `claude/SKILL.md` (Claude-specific umbrella) |
| `codex/SKILL.md.tmpl` | same with `--host codex` | `codex/SKILL.md` (Codex-specific umbrella) |

## SKILL.md Hash Inventory (For Drift Detection)

Hash inventory is not currently generated by the auto-extractor (it expects a centralized `templates/manifests/` layout). To produce one:

```bash
# From repo root
find . -maxdepth 2 -name "SKILL.md" -not -path "./node_modules/*" \
  | sort \
  | xargs -I{} sh -c 'echo "$(sha256sum "{}" | cut -d" " -f1)  {}"' \
  > .agents/research/gstack/golden/skill-md-hashes.txt
```

Run after each `bun run gen:skill-docs` to detect template drift. Commit hash file as the golden fixture.

## Consumer Notes

- A clone effort that copies `<skill>/SKILL.md` instead of `<skill>/SKILL.md.tmpl` will diverge on the next `gen:skill-docs` run.
- Per-host overrides (`<skill>/<host>/SKILL.md`) are intentional shape ‚Äî do not flatten.
- `package.json` `bin` entries point at `dist/` paths that don't exist until `bun run build`. A pre-built tarball is the install-friendly artifact for end users; the repo is the dev surface.
