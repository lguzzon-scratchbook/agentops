# Pre-Mortem Check: Duplicated contract scanners drift when a new accepted syntax is added to one validator without paired fixture coverage in the mirror validator.

Prevent this known failure mode:

- Pattern: Duplicated contract scanners drift when a new accepted syntax is added to one validator without paired fixture coverage in the mirror validator.
- Ask: When a CI gate and smoke test both enforce the same contract, did this change update every recognizer and add a fixture for the newly accepted syntax form?
- Do: For mirrored validators, add or update paired fixtures for each accepted syntax representation and cite both validation commands in the PR proof.
