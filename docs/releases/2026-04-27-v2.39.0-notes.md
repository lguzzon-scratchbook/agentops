## Highlights

AgentOps v2.39.0 is the daemon and CLI architecture release. It lands the
opt-in `agentopsd` control plane: a local always-on runtime with a durable
ledger, job queue, HTTP API, worker supervisor, event stream, projections,
mutation tokens, scheduling, and operator commands. Foreground RPI, Dream,
wiki/forge, and plans flows still work, but they now have a daemon migration
path instead of each command owning runtime state in isolation.

This release also refactors a large portion of the Go CLI into clearer internal
packages for daemon state, agent workers, GasCity integration, scheduling,
evaluation, path resolution, lifecycle, and knowledge surfaces. The command
surface grew accordingly: daemon status/events/jobs/soak, schedule management,
agent-state health checks, skill health checks, edit-scope controls, eval
substrate commands, pool reindexing, and daemon-backed RPI/Dream/plans entry
points.

## What's New

- **AgentOps daemon runtime** - `ao daemon run`, `ready`, `status`, `events`,
  `jobs`, `service`, and `soak` expose the new `agentopsd` control plane for
  local jobs, health, readiness, event tailing, queue inspection, cancellation,
  worker execution, and product proof runs.
- **Daemon-backed workflows** - RPI and Dream gained explicit `--daemon-submit`
  paths, `ao rpi status --daemon` can read daemon projections, `ao plans sync
  --via-daemon` can route through `plans.projection`, and wiki/forge work can
  execute through daemon worker policies.
- **Native scheduling** - `.agents/schedule.yaml`, `ao schedule`, daemon
  `--schedule-file`, cron parsing, backpressure, schedule mutation routes, and
  recurrence ledger events make recurring Dream/wiki/RPI work a daemon-owned
  primitive.
- **Worker and factory substrate** - AgentWorker, GasCity, CLI-fallback workers,
  cgroup caps, routing policy guardrails, scoped mutation tokens, factory
  lifecycle projections, and yield contracts establish the first safe
  parallel-agent control-plane slice.
- **CLI refactor and expansion** - new internal packages split daemon, worker,
  GasCity, schedule, eval, path, lifecycle, harvest, and knowledge logic out of
  the monolithic command layer while adding `ao agents`, `ao skills check`, `ao
  scope`, `ao eval`, `ao pool reindex`, and related command coverage.
- **Eval and release gates** - deterministic `ao eval` primitives, scorecards,
  baseline audit, canary suites, contract canaries, command-surface checks, and
  CI/nightly hardening moved the release process onto stronger local and hosted
  proof.
- **Codex and hooks refresh** - the Codex native plugin was refreshed for
  GPT-5.5/current marketplace metadata, hooks were reworked around managed/JIT
  runtime context, and hook output parity is now linted across Claude/Codex
  surfaces.

## All Changes

### Added

- `agentopsd`, with a durable `.agents/daemon` ledger, local HTTP API, queue,
  event stream, projections, health/readiness endpoints, mutation tokens, worker
  supervisor, service-plan scaffolding, and product soak scenarios.
- Daemon job families and projections for `rpi.run`, `rpi.phase`, `dream.run`,
  `dream.stage`, `wiki.forge`, `wiki.build`, `openclaw.snapshot`, and
  `plans.projection`.
- Daemon command surface: `ao daemon run`, `ready`, `status`, `events tail`,
  `jobs list/show/submit/wait/cancel`, `service install`, and `soak`.
- Scheduling surface: `.agents/schedule.yaml`, `ao schedule add/list/run/remove`,
  schedule schema validation, cron/backpressure controls, schedule mutation
  routes, and recurrence supervisor tests.
- Agent worker substrate: GasCity API/SSE adapters, CLI fallback workers,
  worker schemas, quarantine, process cleanup, Linux cgroup caps, and routing
  policy validation.
- Factory/control-plane contracts for worker slots, worktree ownership,
  validation state, manual merge disposition, lifecycle telemetry, and yield
  ledgers.
- CLI surfaces for `ao agents inspect/lint/doctor`, `ao skills check`, `ao
  scope`, `ao eval task/cleanup/suite/coverage/baseline-audit`, `ao pool
  reindex`, `ao watch`, and daemon-backed plan synchronization.
- Deterministic eval substrate, scorecards, runtime adapters, coverage reports,
  baseline A/B, context-packet A/B, retrieval/file-backed backends, and canary
  suites.
- Dream finding-generator sidecars, aggregation, external-watchlist output, and
  end-user coverage fitness gates.
- `.agents/` write-surface contract, lint gate, no-tracked-`.agents` policy,
  and production-code smoke coverage.

### Changed

- CLI internals were reorganized around focused packages for daemon state,
  worker execution, GasCity, schedule parsing, eval, path resolution, lifecycle,
  harvest, OpenClaw, LLM wiki execution, safety, and quality checks.
- Foreground RPI, Dream, plans, and wiki/forge flows gained opt-in daemon
  submission/read paths while retaining compatibility fallbacks during the
  migration window.
- Hooks were re-architected around JIT context and a managed runtime backend,
  with Claude/Codex PreToolUse output parity checks and quieter Codex
  session-start behavior.
- Path handling moved toward shared Go/shell state-path resolvers
  (`cli/internal/paths`, `lib/ao-paths.sh`) to reduce scattered `.agents` path
  logic.
- Codex runtime packaging was refreshed for GPT-5.5, marketplace metadata,
  `$skill` notation, native hook installation, generated manifests, and reduced
  skill-catalog context.
- CI/local release gates now cover daemon product proofs, contract canaries,
  eval baselines, command/test pairing, Codex runtime sections, pre-push wiring,
  release audit artifacts, Windows smoke, advisory policy, and nightly
  knowledge-cycle dedupe.
- Release and operator docs now describe the daemon migration, scheduling
  contract, GasCity readiness, OpenClaw consumer paths, local compute routing,
  and the no-tracked-`.agents` policy.

### Fixed

- Daemon durability and state-machine edges: fsync propagation, orphan temp
  sweeps, snapshot directory sync, idempotency-key dedupe, queue cancellation,
  recurrence recompute, projection deep-copy/nil-safety, claimed job queue
  depth, and terminal projection precedence.
- Daemon API and execution hardening: body caps, event limits, unknown-cursor
  errors, path containment, schedule name traversal, UTF-8 truncation, oversized
  ledger lines, worker context cancellation, and daemon Dream/wiki execution
  through the supervisor.
- CLI hygiene: JSON output validity, UTF-8 truncation, schedule prompt behavior
  under tests, command catalog regeneration, Cobra docs conformance, hidden
  command surface checks, and command/test pairing.
- Hook/runtime drift: Codex skill chaining defaults, native hook manifest
  installation, noisy session-start output, hook output schema, edit/write hash
  audit hooks, intent-echo bypass for team runners, and lifecycle guard coverage.
- CI/nightly failures: eval advisory fixtures, baseline-audit drift-only
  behavior, Windows smoke paths, shellcheck/pre-push gate regressions,
  knowledge-cycle dormant-corpus handling, and release audit validation.
