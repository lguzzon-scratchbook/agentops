## Highlights

AgentOps v2.39.0 makes repo memory easier to inspect and safer to release. New
`ao agents` commands show, lint, and diagnose the `.agents/` write surfaces,
Dream can turn overnight findings into structured follow-up candidates, and the
release gate now validates the current work-queue contract instead of failing on
stale enum assumptions. The Codex package was also refreshed for GPT-5.5, with
runtime drift fixes across skill invocation, hooks, and metadata.

## What's New

- **Agent state health commands** - `ao agents inspect`, `ao agents lint`, and
  `ao agents doctor` help operators understand and validate the `.agents/`
  memory surfaces before they become release risk.
- **Structured overnight follow-up** - Dream can record and aggregate finding
  generator sidecars, including external-watchlist candidates for work that
  needs human review outside the normal queue.
- **Stronger work-queue metadata** - Work items now carry validated status,
  prerequisite, and deduplication fields end to end.
- **Goals filtering** - `ao goals measure --exclude-tag` lets long-cycle corpus
  state be excluded from selected goal measurements.
- **Codex package refresh** - Codex skills and metadata were updated for GPT-5.5
  and the current marketplace schema.

## All Changes

### Added

- Agent state health commands for inspecting, linting, and diagnosing `.agents/`
  write surfaces
- Dream finding-generator sidecars, aggregation, and external-watchlist output
- First-class validated metadata for work-item status, prerequisites, and
  deduplication
- Goal measurement filtering by excluded tag
- Write-surface contract and smoke coverage for repo memory
- Nested artifact harvesting with real rig metadata

### Changed

- Codex runtime package refreshed for GPT-5.5 and current marketplace metadata
- Hook runtime internals refactored around a managed backend
- Documentation index, operator guide, comparison docs, and finding-generator
  RFC updated
- Release gates now cover more of the work-queue, Codex, command/test, and
  audit-artifact surfaces
- Bootstrap now recommends `bd` installation instead of attempting automatic
  installation

### Fixed

- CLI JSON and UTF-8 truncation bugs
- Docs link drift and CLI command catalog drift
- Codex skill chaining, hook manifest, session-start, and `$skill` notation
  drift
- Harvest path handling and scanner false positives that could block release
  gates
- RPI, triage, coverage parsing, and release-gate regressions found during
  local validation
