## Why This Patch Exists

v2.41.0 was tagged on `eb119f48` while three CI `Validate` checks were already red. They were invisible to the release validation because the path-filtered CI marks untriggered jobs `skipped` (not `failed`), and the intervening commit only touched `scripts/extract-release-notes.sh` — so the skill/registry/canary jobs never ran. The failures only surfaced when a later commit touched skill paths.

v2.41.1 is the CI-green re-cut. The operator chose a patch release over force-moving the v2.41.0 tag so v2.41.0 stays as immutable history.
